# 安装配置指南

本文说明运行 `asana-openspec-java-workflow` 需要安装或启用的外部工具。插件本身只提供 Codex skills 和模板，不内置 OpenSpec、CodeGraph、MySQL MCP。

## 最低版本

| 工具 | 最低要求 | 说明 |
|---|---|---|
| Codex | 支持 plugins / skills / AGENTS.md 的版本 | 以团队当前 Codex 版本为准 |
| Node.js | 20.19.0+ | OpenSpec 要求 |
| OpenSpec | `@fission-ai/openspec@latest` | 团队统一安装 latest |
| CodeGraph | `@colbymchenry/codegraph` 当前版本 | 用于代码图谱 |
| MySQL MCP | 选定 server 当前稳定版本 | 例如 `@benborla29/mcp-server-mysql` |
| JDK / Maven / Gradle | 跟随项目仓库 | 不在工具包里统一 |

## 必需工具

### Codex

团队成员需要能在目标代码仓库里使用 Codex，并能读取项目 `AGENTS.md`。

建议：

- 每个 Java 仓库都放一个项目级 `AGENTS.md`。
- 把 `assets/templates/AGENTS模板.md` 内容合并进去。
- 如果团队多人使用，优先把规则提交到仓库，而不是只放个人本地配置。

### Node.js

OpenSpec 当前要求 Node.js 20.19.0 或更高版本。

检查：

```powershell
node --version
```

如果版本不够，先升级 Node.js。

### OpenSpec

OpenSpec 用来生成和维护变更文档：

```text
openspec/changes/<change-id>/
  proposal.md
  design.md
  tasks.md
  specs/
```

安装：

```powershell
npm install -g @fission-ai/openspec@latest
```

验证：

```powershell
openspec --version
```

在项目仓库初始化：

```powershell
cd <your-java-project>
openspec init
```

推荐启用 expanded workflow：

```powershell
openspec config profile
openspec update
```

### OpenSpec 命令使用时机

| 命令 | 何时使用 |
|---|---|
| `/opsx:new <id>` | PRD 确认后，创建新 change |
| `/opsx:continue` | 继续编辑当前 change |
| `/opsx:ff` | 需求清楚时，快速填充 proposal/design/tasks |
| `/opsx:apply` | Plan Review 通过后，应用 change 到代码 |
| `/opsx:verify` | PR 合并后验证 OpenSpec 状态 |
| `/opsx:archive` | verify 通过后归档 change |

## 推荐工具

### CodeGraph

CodeGraph 用来生成代码知识图谱，减少 AI 乱定位代码。

安装：

```powershell
npx @colbymchenry/codegraph
```

非交互方式示例：

```powershell
codegraph install --target=auto --location=local --yes
```

初始化项目：

```powershell
cd <your-java-project>
codegraph init -i
```

初始化后项目会有 `.codegraph/`。之后按 `codegraph-context-guard` 规则使用：

```text
修改前：查入口、调用链、影响面
修改后：PR 前复查 impact
```

### Asana Connector

Asana 是可选需求跟踪入口之一，用于负责人、优先级、排期、状态流转和跨团队协作。没有 Asana 时，也可以先通过用户对话、会议纪要、线上问题或技术债想法生成 PRD 草稿；需要团队追踪时再后补 Asana。

配置方式：

- 在 Codex App / Connectors 里启用 Asana。
- 确认 Codex 能读取任务标题、描述、评论、附件。
- 不建议第一版自动改 Asana 状态；先人工流转。

字段模板：

```text
assets/templates/Asana字段模板.md
```

### GitHub Connector

GitHub 用于 PR、Review、CI 状态。

配置方式：

- 在 Codex App / Connectors 里启用 GitHub。
- 授权目标组织和仓库。
- 确认 Codex 能读取 PR、文件 diff、CI 状态。

第一版建议：

- Codex 生成 PR 描述和 Review 清单。
- 人工开 PR 或人工确认后再开 PR。

## 可选工具

### MySQL MCP

MySQL MCP 用来查表结构、验证 SQL、做受控数据写入。

可选实现之一：

```powershell
npm install -g @benborla29/mcp-server-mysql
```

Codex CLI 配置示例，按实际环境调整：

```powershell
codex mcp add mcp_server_mysql `
  --env MYSQL_HOST="127.0.0.1" `
  --env MYSQL_PORT="3306" `
  --env MYSQL_USER="ai_agent" `
  --env MYSQL_PASS="<from-secret-manager>" `
  --env MYSQL_DB="your_database" `
  --env ALLOW_INSERT_OPERATION="true" `
  --env ALLOW_UPDATE_OPERATION="true" `
  --env ALLOW_DELETE_OPERATION="true" `
  -- npx -y @benborla29/mcp-server-mysql
```

如果选择的 MySQL MCP server 对 DDL 有额外开关，例如只读事务限制，需要按该 server 文档开启对应配置。无论 server 是否允许，团队流程仍必须执行 `mysql-db-guard`：

```text
DELETE：先 SELECT 预览，再明确授权
CREATE / 非破坏性 ALTER：强确认后执行
DROP / TRUNCATE / ALTER DROP：默认拒绝
```

AI 专用账号授权示例：

```sql
CREATE USER 'ai_agent'@'%' IDENTIFIED BY '<use-secret-manager>';

GRANT SELECT, SHOW VIEW, EXECUTE
ON your_db.* TO 'ai_agent'@'%';

GRANT INSERT, UPDATE, DELETE
ON your_db.* TO 'ai_agent'@'%';

GRANT CREATE, ALTER
ON your_db.* TO 'ai_agent'@'%';

FLUSH PRIVILEGES;
```

不要授予：

```sql
DROP
TRUNCATE
GRANT OPTION
SUPER
FILE
PROCESS
SHUTDOWN
RELOAD
```

## Java 项目建议

### JDK / Maven / Gradle

每个项目按仓库现有规则配置，不在本工具包里统一版本。

建议：

- 优先使用项目 wrapper：`mvnw.cmd` / `gradlew.bat`。
- 遵守项目本地 formatter、Checkstyle、Spotless、PMD、Sonar。
- 构建失败时使用 `java-build-fix`，不要跳过测试伪造成功。

### 测试框架

优先使用项目已有测试栈。

常见组合：

```text
JUnit 5
Mockito
AssertJ
Spring Boot Test
MockMvc / WebMvcTest
Testcontainers 或项目现有测试库方案
```

行为变更必须有测试，或说明不能自动化原因。

## 第一次安装顺序

推荐顺序：

```text
1. 按 团队安装指南.md clone plugin 到本地插件源码目录
2. 创建 personal marketplace，并执行 `codex plugin add asana-openspec-java-workflow@personal`
3. 启用 Codex
4. 合并 AGENTS模板.md 到项目 AGENTS.md
5. 安装 Node.js 20.19+
6. 安装并初始化 OpenSpec
7. 可选：安装并初始化 CodeGraph
8. 可选：启用 Asana/GitHub Connector
9. 可选：配置 MySQL MCP 和 ai_agent 账号
10. 选一条小需求试跑
```

## 最小可运行组合

如果只想先试流程：

```text
Codex + AGENTS.md + OpenSpec
```

Asana Connector 是团队需求跟踪增强项，不是启动 PRD / OpenSpec 流程的硬依赖。

如果要减少 AI 乱定位：

```text
Codex + AGENTS.md + OpenSpec + CodeGraph
```

如果要让 AI 查库/验证 SQL：

```text
Codex + AGENTS.md + OpenSpec + MySQL MCP + ai_agent
```

如果要团队完整试点：

```text
Codex + Asana + GitHub + OpenSpec + CodeGraph + MySQL MCP
```

## 工具失败处理

详细见 `工具失败处理.md`。原则：

- OpenSpec 失败：停止实现，先修 change 状态。
- CodeGraph 失败：降级到 `rg` / 文件读取 / 人工 Review。
- MySQL MCP 失败：不绕过安全规则，改为生成 SQL 和人工执行步骤。
- 构建/测试失败：使用 `java-build-fix`，不跳过测试伪造成功。
