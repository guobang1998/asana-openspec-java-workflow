# 多入口需求 + OpenSpec Java Workflow 流程图

## 总览：先分流，再执行

<!-- route-priority: setup > wayfinder > brainstorming -->

```mermaid
flowchart LR
  A["需求入口<br/>Asana / 用户对话 / 会议纪要 / 线上问题 / 技术债"] --> S0{"代码规范地图可用?"}
  S0 -- "否" --> S1["setup-workflow<br/>只读发现"] --> R{"入口类型判断"}
  S0 -- "是 / 不适用" --> R
  R -- "跨会话大目标 / 路径不清" --> W["Wayfinder map<br/>只画图"]
  W --> W1["一次只推进一张<br/>frontier ticket"] --> R
  R -- "明确需求" --> B["PRD Writer<br/>生成或完善 PRD"]
  R -- "未命中 Wayfinder 的<br/>模糊需求 / 新功能探索 / 方案分歧" --> C1["轻量澄清或<br/>Superpowers brainstorming"]
  R -- "线上问题" --> P["根因定位<br/>确认处理模式"]
  R -- "目标明确的技术债 / 重构" --> T["Discovery / RFC<br/>必要时大重构流程"]
  C1 --> B
  P --> B
  T --> E{"是否大重构?"}
  B --> C{"PRD 是否清楚并确认?"}
  C -- "否" --> D["补待确认问题<br/>回到来源或 Asana 评论"] --> B
  C -- "是" --> E

  F --> H["CodeGraph / 实现 / 测试<br/>分段审核 / 必要时高风险复查 / PR Gate"]
```

硬门槛：

- 模糊需求不能直接实现。
- brainstorming 不能直接替代 PRD / OpenSpec。
- PRD / OpenSpec 未确认，不进入行为变更实现。

## 小需求流程：PRD 到单个 OpenSpec change

```mermaid
flowchart TB
  A["需求入口<br/>Asana / 对话 / 会议 / 问题 / 技术债"] --> R{"是否清楚?"}
  R -- "模糊" --> R1["轻量澄清或<br/>Superpowers brainstorming"]
  R -- "清楚" --> B["PRD md<br/>docs/prd/*.prd.md"]
  R1 --> B
  B --> C{"PRD 是否清楚?"}
  C -- "否" --> C1["待确认问题<br/>回到来源或 Asana 评论"] --> B
  C -- "是" --> C2["人工确认 PRD"]
  C2 --> D0["既有规格检查<br/>specs / active changes / 历史 PRD"]
  D0 --> D["OpenSpec change<br/>proposal/design/tasks/specs"]
  D --> E["影响面定位<br/>CodeGraph 优先<br/>rg/read 降级"]
  E --> F["Coding Discipline<br/>假设/范围/不做/验证"]
  F --> G["Plan Review<br/>任务拆分/风险/测试策略/回滚"]
  G --> G0["分段审核基础判断<br/>风险等级/是否建议高风险复查"]
  G0 --> G1{"实现复杂 / 多文件多步骤?"}
  G1 -- "是" --> G2["可选 Superpowers writing-plans<br/>拆文件/步骤/测试/提交"]
  G1 -- "否" --> H["Codex 实现<br/>按 tasks.md 小步推进"]
  G2 --> H
  H --> I["Java Test Strategy<br/>执行单测/集成/手动验证"]
  I --> J["CodeGraph impact<br/>改动影响面复查"]
  J --> K0["PR 前分段审核<br/>B/A/大重构补扩展字段"]
  K0 --> K1{"是否需要高风险复查?"}
  K1 -- "是" --> K2["高风险复查<br/>范围/边界/风险/证据"]
  K1 -- "否" --> K["Review Gates<br/>Java/Security/SQL Performance/MySQL/Quality"]
  K2 --> K
  K --> L{"是否通过?"}
  L -- "业务理解错" --> B
  L -- "方案设计错" --> D
  L -- "实现 bug" --> H
  L -- "测试不足" --> I
  L -- "是" --> M["开 PR<br/>需求来源 + OpenSpec + 测试结果"]
  M --> N["合并"]
  N --> O["OpenSpec verify/archive"]
  O --> P["完成后记录<br/>耗时/Review/bug/回退/定位准确性"]
  P --> Q["有 Asana 时标记完成"]

  class K,L,M review;
  class N,O,P,Q done;
```

## 大重构流程：Epic + RFC + 可选多 Agent 协作

```mermaid
flowchart TB
  A["重构入口<br/>Asana Epic / 技术债 / 架构治理"] --> WF{"Destination / 范围 / 首批决策清楚?"}
  WF -- "否" --> WF1["Wayfinder map<br/>一次只解决一张 frontier ticket"] --> WF
  WF -- "是" --> B["CodeGraph Discovery<br/>入口/调用链/callers/callees/impact"]
  B --> C["重构 RFC<br/>docs/refactor/*/重构RFC.md"]
  C --> D["测试基线<br/>Characterization Tests"]
  D --> E["Phase 拆分<br/>0 Discovery / 1 Adapter / 2 Migrate / 3 Switch / 4 Cleanup"]
  E --> E0["实现前边界声明<br/>A 类风险映射"]
  E0 --> E01["分段审核基础判断<br/>按风险判断复查"]
  E01 --> E1{"是否启用多 agent?"}
  E1 -- "否" --> S["单 Codex agent<br/>按 phase 分阶段推进"]
  E1 -- "是" --> E2["确认 worker 数量/角色/并发/Claude"]
  E2 --> T[".workflow-team/<change-id><br/>config/dependencies/status/tasks/results"]

  T --> W1["Discovery Worker<br/>影响面/调用链"]
  T --> W2["OpenSpec Worker<br/>proposal/design/tasks/specs"]
  T --> W3["Implementation Worker<br/>独立 worktree"]
  T --> W4["Test Worker<br/>测试/回归/证据"]
  T --> W5["Claude Reviewer 可选<br/>$cc:review / adversarial"]

  W1 --> J["result.md + result-metadata.yaml"]
  W2 --> J
  W3 --> J
  W4 --> J
  W5 --> J

  S --> P1["Phase OpenSpec changes<br/>小 PR 串行"]
  J --> K["Leader 合并<br/>依赖/锁/冲突记录"]
  K --> P1
  P1 --> P2["Phase 出口分段审核<br/>必要时高风险复查"]
  P2 --> K1["回滚验证<br/>测试环境验证或说明原因"]
  K1 --> L["兼容/灰度/回滚<br/>adapter / facade / feature flag"]
  L --> M["删除旧代码<br/>只在最后阶段"]
  M --> N["复盘 + archive"]
  N --> O["有 Asana Epic 时更新状态"]

  classDef rfc fill:#14b8a6,color:#042f2e,stroke:#0f766e;
  classDef phase fill:#f59e0b,color:#111827,stroke:#b45309;
  classDef pr fill:#f97316,color:#111827,stroke:#c2410c;
  classDef done fill:#16a34a,color:#fff,stroke:#15803d;

  class A epic;
  class WF,WF1 discover;
  class B discover;
  class C,D rfc;
  class E,E0,E01,E1,E2,T,W1,W2,W3,W4,W5,J phase;
  class S,P1,P2,K,K1,L,M pr;
  class N,O done;
```
