# 团队安装指南

本文说明如何把 `asana-openspec-java-workflow` 分发给团队成员。

## 推荐方式

把本工具包放到团队 Git 仓库。团队成员 clone 到自己的本地插件源码目录，再通过 Codex personal marketplace 安装。

## 维护者：发布工具包

在工具包目录初始化 Git：

```powershell
cd E:\asana-openspec-java-workflow
git init
git add .
git commit -m "init asana-openspec-java-workflow"
```

添加远程仓库并推送：

```powershell
git remote add origin <repo-url>
git push -u origin main
```

建议仓库名：

```text
asana-openspec-java-workflow
```

## 团队成员：安装插件

### Windows

```powershell
git clone <repo-url> "$env:USERPROFILE\plugins\asana-openspec-java-workflow"
```

如果目录已存在：

```powershell
cd "$env:USERPROFILE\plugins\asana-openspec-java-workflow"
git pull
```

创建或确认 personal marketplace：

```powershell
$dir = "$env:USERPROFILE\.agents\plugins"
New-Item -ItemType Directory -Force -Path $dir | Out-Null
$json = @'
{
  "name": "personal",
  "interface": {
    "displayName": "Personal"
  },
  "plugins": [
    {
      "name": "asana-openspec-java-workflow",
      "source": {
        "source": "local",
        "path": "./plugins/asana-openspec-java-workflow"
      },
      "policy": {
        "installation": "AVAILABLE",
        "authentication": "ON_INSTALL"
      },
      "category": "Productivity"
    }
  ]
}
'@
$utf8NoBom = New-Object System.Text.UTF8Encoding($false)
[System.IO.File]::WriteAllText("$dir\marketplace.json", $json + [Environment]::NewLine, $utf8NoBom)
```

安装插件：

```powershell
codex plugin add asana-openspec-java-workflow@personal
codex plugin list
```

如果 WindowsApps 版 `codex.exe` 提示 `Access is denied`，改用：

```powershell
& "$env:USERPROFILE\.codex\npm-global\codex.cmd" plugin add asana-openspec-java-workflow@personal
& "$env:USERPROFILE\.codex\npm-global\codex.cmd" plugin list
```

安装或更新后，重启 Codex App，或在目标仓库新开一个 Codex 会话。这样当前会话才能重新加载插件 skills。

### macOS / Linux

```bash
git clone <repo-url> ~/plugins/asana-openspec-java-workflow
```

如果目录已存在：

```bash
cd ~/plugins/asana-openspec-java-workflow
git pull
```

创建或确认 personal marketplace：

```bash
mkdir -p ~/.agents/plugins
cat > ~/.agents/plugins/marketplace.json <<'JSON'
{
  "name": "personal",
  "interface": {
    "displayName": "Personal"
  },
  "plugins": [
    {
      "name": "asana-openspec-java-workflow",
      "source": {
        "source": "local",
        "path": "./plugins/asana-openspec-java-workflow"
      },
      "policy": {
        "installation": "AVAILABLE",
        "authentication": "ON_INSTALL"
      },
      "category": "Productivity"
    }
  ]
}
JSON
```

安装插件：

```bash
codex plugin add asana-openspec-java-workflow@personal
codex plugin list
```

安装或更新后，重启 Codex App，或在目标仓库新开一个 Codex 会话。这样当前会话才能重新加载插件 skills。

## 团队成员：项目接入

只 clone plugin 还不够。每个 Java 项目还需要接入项目级规则。

### 1. 合并 AGENTS.md

把下面文件内容合并到目标项目根目录的 `AGENTS.md`：

```text
~/plugins/asana-openspec-java-workflow/assets/templates/AGENTS模板.md
```

Windows 对应路径：

```text
%USERPROFILE%\plugins\asana-openspec-java-workflow\assets\templates\AGENTS模板.md
```

如果项目没有 `AGENTS.md`，直接新建。

### 2. 初始化 OpenSpec

```bash
npm install -g @fission-ai/openspec@latest
cd <your-java-project>
openspec init
openspec config profile
openspec update
```

### 3. 可选：初始化 CodeGraph

```bash
npx @colbymchenry/codegraph
cd <your-java-project>
codegraph init -i
```

### 4. 可选：配置 MySQL MCP

见：

```text
安装配置指南.md
assets/templates/MySQL数据库安全模板.md
```

## 使用方式

开始试跑前，可先在 Codex 里确认插件是否已加载：

```text
请列出当前会话可用的 asana-openspec-java-workflow skills。
```

如果 Codex 提示没有 `coding-discipline`、`prd-writer` 等裸名 skill，先确认是否存在带前缀名称，例如 `asana-openspec-java-workflow:coding-discipline`、`asana-openspec-java-workflow:prd-writer`。如果带前缀名称也没有，执行 `codex plugin list`，确认 `asana-openspec-java-workflow@personal` 是 `installed, enabled`。确认安装后仍不可用，重启 Codex 或新开会话。项目 `AGENTS.md` 中的规则可作为兜底，试跑不需要因此中断。

在 Codex 中说：

```text
请按 asana-openspec-java-workflow 多入口需求路由处理这个需求。它可能来自 Asana、用户对话、会议纪要、线上问题或技术债想法；如果需求模糊，先澄清，不要写代码。
```

或：

```text
请使用 prd-writer，把这个需求整理成 PRD md。信息不够先列待确认问题，不要写代码。
```

没有 Asana 的模糊需求：

```text
这不是 Asana 任务，是会议或对话里提到的想法。请先做轻量澄清；如果范围不清或方案分歧较大，用 brainstorming 澄清后再生成 PRD 草稿，不要写代码。
```

PRD 确认后：

```text
PRD 已确认。请创建 OpenSpec change，并按 asana-openspec-java-workflow 推进 design/tasks/specs。
```

## 更新方式

维护者更新工具包后，团队成员执行：

Windows：

```powershell
cd "$env:USERPROFILE\plugins\asana-openspec-java-workflow"
git pull
codex plugin add asana-openspec-java-workflow@personal
```

macOS / Linux：

```bash
cd ~/plugins/asana-openspec-java-workflow
git pull
codex plugin add asana-openspec-java-workflow@personal
```

如果项目里的 `AGENTS.md` 模板也更新了，需要手动同步到项目。

## 注意事项

- `~/plugins/asana-openspec-java-workflow` 是本地插件源码目录。
- `~/.agents/plugins/marketplace.json` 是 Codex personal marketplace 入口。
- `~/.codex/plugins/cache` 是 Codex 安装缓存，不建议把源码 clone 到这里。
- 项目 `AGENTS.md` 用于保证仓库级规则可被 Codex 读取。
- OpenSpec 必须在每个项目仓库单独初始化。
- CodeGraph 也必须在每个项目仓库单独初始化。
- 不要把数据库密码提交到 Git。
- 不要把团队私有 Asana/GitHub token 提交到 Git。

## 最小试点

如果团队第一次试用，建议只做：

```text
1. clone plugin
2. 创建 personal marketplace 并执行 `codex plugin add asana-openspec-java-workflow@personal`
3. 合并 AGENTS.md
4. 初始化 OpenSpec
5. 选一条小需求跑完
```

建议先试跑两类小需求：

1. 已有 Asana 的明确小需求：验证 Asana -> PRD -> OpenSpec -> PR Gate。
2. 没有 Asana 的模糊需求：验证 brainstorming / 轻量澄清 -> PRD 草稿 -> OpenSpec -> PR Gate。

跑通 3 条需求后，再统一接入 CodeGraph、MySQL MCP 和自动化状态同步。
