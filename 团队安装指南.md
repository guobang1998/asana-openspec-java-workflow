# 团队安装指南

本文说明如何把 `asana-openspec-java-workflow` 分发给团队成员。

## 推荐方式

把本工具包放到团队 Git 仓库。团队成员 clone 到自己的 Codex plugins 目录。

## 团队成员：安装插件

### Windows

```powershell
git clone <repo-url> "$env:USERPROFILE\.codex\plugins\asana-openspec-java-workflow"
```

如果目录已存在：

```powershell
cd "$env:USERPROFILE\.codex\plugins\asana-openspec-java-workflow"
git pull
```

### macOS / Linux

```bash
git clone <repo-url> ~/.codex/plugins/asana-openspec-java-workflow
```

如果目录已存在：

```bash
cd ~/.codex/plugins/asana-openspec-java-workflow
git pull
```

## 团队成员：项目接入

只 clone plugin 还不够。每个 Java 项目还需要接入项目级规则。

### 1. 合并 AGENTS.md

把下面文件内容合并到目标项目根目录的 `AGENTS.md`：

```text
~/.codex/plugins/asana-openspec-java-workflow/assets/templates/AGENTS模板.md
```

Windows 对应路径：

```text
%USERPROFILE%\.codex\plugins\asana-openspec-java-workflow\assets\templates\AGENTS模板.md
```

如果项目没有 `AGENTS.md`，直接新建。

### 2. 初始化 OpenSpec

```bash
npm install -g @fission-ai/openspec@latest
cd <your-java-project>
openspec init
openspec config profile
openspec update
```

### 3. 可选：初始化 CodeGraph

```bash
npx @colbymchenry/codegraph
cd <your-java-project>
codegraph init -i
```

### 4. 可选：配置 MySQL MCP

见：

```text
安装配置指南.md
assets/templates/MySQL数据库安全模板.md
```

## 使用方式

在 Codex 中说：

```text
请按 asana-openspec-java-workflow 流程处理这个 Asana 需求。
```

或：

```text
请使用 prd-writer，把这个需求整理成 PRD md。信息不够先列待确认问题，不要写代码。
```

PRD 确认后：

```text
PRD 已确认。请创建 OpenSpec change，并按 asana-openspec-java-workflow 推进 design/tasks/specs。
```

## 更新方式

维护者更新工具包后，团队成员执行：

Windows：

```powershell
cd "$env:USERPROFILE\.codex\plugins\asana-openspec-java-workflow"
git pull
```

macOS / Linux：

```bash
cd ~/.codex/plugins/asana-openspec-java-workflow
git pull
```

如果项目里的 `AGENTS.md` 模板也更新了，需要手动同步到项目。

## 注意事项

- 插件目录用于 Codex skills 和模板分发。
- 项目 `AGENTS.md` 用于保证仓库级规则可被 Codex 读取。
- OpenSpec 必须在每个项目仓库单独初始化。
- CodeGraph 也必须在每个项目仓库单独初始化。
- 不要把数据库密码提交到 Git。
- 不要把团队私有 Asana/GitHub token 提交到 Git。

## 最小试点

如果团队第一次试用，建议只做：

```text
1. clone plugin
2. 合并 AGENTS.md
3. 初始化 OpenSpec
4. 选一条小需求跑完
```

跑通 3 条需求后，再统一接入 CodeGraph、MySQL MCP 和自动化状态同步。
