# 第一阶段规范入口与四轴审查实施计划

> **供执行 agent 使用：** 按本计划执行时，先运行 `scripts/验证第一阶段工作流.ps1`，确认缺口存在；每完成一项后重跑校验。

**目标：** 让 Java 仓库在首次需求推进时能发现本地规范，生成可复用的代码规范地图，并以四轴输出 Review 与 PR Gate 结论。

**架构：** setup 只读发现仓库事实，用户确认后才写代码规范地图；主交付流程自动加载地图。Review 与 PR Gate 按四轴呈现，但继续沿用原有 Java、SQL、DB、安全和测试门禁。

**技术栈：** Codex plugin、Markdown skill、PowerShell 静态验收。

---

**边界：** 本阶段不实现 Wayfinder、Frontier Ticket、TDD/Debug 新 skill，也不替换既有 Java、SQL、DB 或安全门禁。

## 文件职责

| 路径 | 职责 |
|---|---|
| `skills/setup-workflow/SKILL.md` | 发现仓库事实，给出受确认保护的写入计划。 |
| `assets/templates/代码规范地图模板.md` | 规范来源、构建命令、分层、接口字段权威和风险门禁的唯一项目化索引。 |
| `skills/asana-openspec-delivery/SKILL.md` | 新仓库或缺少规范地图时，先进入 setup。 |
| `skills/java-backend-review/SKILL.md` | 输出 Standards、Spec、Tests、Risk Gates 四轴。 |
| `skills/pr-quality-gate/SKILL.md` | 按四轴汇总 PR 结论，保留原有阻断条件。 |
| `README.md` | 让团队成员知道首次接入和使用方式。 |
| `.codex-plugin/plugin.json` | 发布版本与能力声明。 |
| `scripts/验证第一阶段工作流.ps1` | 静态验收新入口和四轴合同。 |
