> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为插件增加 `delivery-convergence-loop`，把计划预检、综合 Review、集中修复和增量复审编排成内部收敛流程，减少重复全量审查和用户中断。

**Tech Stack:** Codex Skill Markdown、Codex plugin manifest、PowerShell 静态/行为验证脚本、Git fixed point、JSON/JSONL 运行状态。

function Require-File([string]$RelativePath) {
    if (-not (Test-Path -LiteralPath (Join-Path $RepoRoot $RelativePath))) {
        $Errors.Add("缺少文件：$RelativePath")
    }
}

function Require-Text([string]$RelativePath, [string]$ExpectedText) {
    $Path = Join-Path $RepoRoot $RelativePath
    if (-not (Test-Path -LiteralPath $Path)) {
        $Errors.Add("缺少文件：$RelativePath")
        return
    }
    $Content = Get-Content -LiteralPath $Path -Raw -Encoding UTF8
    if (-not $Content.Contains($ExpectedText)) {
        $Errors.Add("$RelativePath 缺少约定文本：$ExpectedText")
    }
}

function Require-StructuredBlock(
    [string]$RelativePath,
    [string]$Marker,
    [string]$HeadingPattern,
    [string[]]$RequiredBodyPatterns
) {
    $Path = Join-Path $RepoRoot $RelativePath
    if (-not (Test-Path -LiteralPath $Path)) {
        $Errors.Add("缺少文件：$RelativePath")
        return
    }
    $Content = Get-Content -LiteralPath $Path -Raw -Encoding UTF8
    $MarkerPattern = [regex]::Escape($Marker)
    $SectionPattern = "(?ms)^#{2,4}\\s+$HeadingPattern\\s*\\r?\\n$MarkerPattern\\s*\\r?\\n(?<body>.*?)(?=^#{2,4}\\s+|\\z)"
    $Match = [regex]::Match($Content, $SectionPattern)
    if (-not $Match.Success) {
        $Errors.Add("$RelativePath 缺少标题、marker 相邻的结构化区块：$Marker")
        return
    }
    $Body = $Match.Groups['body'].Value
    foreach ($Pattern in $RequiredBodyPatterns) {
        if ($Body -notmatch $Pattern) {
            $Errors.Add("$RelativePath 的 $Marker 区块正文不完整：$Pattern")
        }
    }
}

`SKILL.md` 必须声明 `交付状态合同.json` 是状态转移唯一机器源；自然语言与 JSON 冲突时测试失败并以 JSON 为准。脚本额外禁止出现以下精确反向 marker：

```text
<!-- no-fix-path: incremental-review-required -->
<!-- required-tests: blocked-may-be-ready -->
<!-- reviewer-mode: writer -->
<!-- p0-p1-waiver: allowed -->
<!-- nested-review-dispatch: allowed -->
```

脚本还要从 JSON 的 `transitions` 按状态名、事件名 ordinal 排序生成 canonical 行：`<STATE> + <EVENT> -> <NEXT>`，与 `SKILL.md` 中 `<!-- generated-state-table:start -->` 和 `<!-- generated-state-table:end -->` 之间内容逐字比较。SKILL 在该区块外不得重复机器状态转移；脚本移除 canonical 区块后，若剩余正文出现任一状态名并同时含 `->` 或 `→`，直接 FAIL。这样错误状态路径不能靠同时保留正确 marker 通过。
