# 工具失败处理

工具失败时不要绕过流程。先记录失败，再降级或暂停。

## OpenSpec 失败

常见场景：

- `openspec init` 失败。
- `/opsx:new` 没生成 change。
- `/opsx:ff` 输出不完整。
- `/opsx:verify` 不通过。
- `/opsx:archive` 失败。

处理：

1. 停止实现，不继续写代码。
2. 记录命令、错误、当前 change-id。
3. 检查 Node.js 版本、OpenSpec 版本、当前目录是否为项目根目录。
4. 修复 OpenSpec 文档或配置。
5. 重新执行失败命令。
6. 仍失败时，把问题写入 Asana 评论或 PR 说明。

## CodeGraph 失败

常见场景：

- 没有 `.codegraph/`。
- 索引过期。
- MCP/CLI 不可用。
- 查不到 symbol。

处理：

1. 说明 CodeGraph 不可用或信息不足。
2. 降级到 `rg` / `find`(bash) 或 `Get-ChildItem`(PowerShell) / 文件读取，手动追踪入口、调用链、影响面。
3. 仍不确定时，依赖人工 Code Review。
4. 在 PR 描述里标注影响面定位方式和不确定点。

## MySQL MCP 失败

常见场景：

- 连接失败。
- 权限不足。
- SQL 被安全规则拦截。
- DELETE/DDL 未确认。

处理：

1. 不绕过 `mysql-db-guard`。
2. 输出 SQL、影响面、回滚方案。
3. 改为人工执行或 DBA 执行。
4. 不把密码写进日志、PRD、OpenSpec 或 PR。

## 构建/测试失败

处理：

1. 使用 `java-build-fix` 收集完整错误。
2. 一次只修一个根因。
3. 重新运行同一命令验证。
4. 不删除测试、不跳过测试、不用 `@Disabled` 伪造通过。

## Asana / GitHub Connector 失败

处理：

1. 手动复制任务链接、PR 链接和关键上下文。
2. 不阻塞代码本地推进，但 PRD、OpenSpec、PR 描述必须补齐链接。
3. 连接恢复后补回 Asana 状态和评论。
