# OpenSpec 检查清单

## 创建前

- [ ] Asana 需求已确认。
- [ ] PRD 已有背景、目标、验收标准。
- [ ] PRD 信息不足时，已输出待确认问题并回 Asana 评论。
- [ ] 已检查相关 `openspec/specs/*`，确认当前系统规格和旧验收标准。
- [ ] 已检查 `openspec/changes/*` active changes，确认没有并行变更冲突。
- [ ] 已检查相关历史 PRD / Asana / archived change，确认本次不是孤立改动。
- [ ] 影响面已确认。
- [ ] 需求边界已写明 `不做什么`。
- [ ] 已判断不是大重构；如果是大重构，已切到 `large-refactor-workflow`。
- [ ] 已确认实现范围，不包含未要求功能或无关重构。
- [ ] 已确认定位方式：CodeGraph / 手动追踪 / 人工 Review。
- [ ] 已确认入口、调用链和影响面。
- [ ] 已判断分段审核风险等级：A 类 / B 类 / C 类。
- [ ] 命中 A 类或明显 B 类风险时，已记录是否建议实现前复查。

## Change 文件

- [ ] `proposal.md` 说明为什么改、改什么。
- [ ] `design.md` 说明方案、风险、回滚。
- [ ] `design.md` 已写明“与既有规格/历史需求的关系”。
- [ ] `tasks.md` 可执行、可勾选。
- [ ] `specs/` 写明行为变化和验收标准。
- [ ] Plan Review 已确认任务拆分、风险点、测试策略和回滚方案。
- [ ] 命中 A 类或明显 B 类风险时，已写实现前边界声明。
- [ ] 命中 A 类风险时，已写高风险链路映射、当前控制方式和剩余风险。

## 实现中

- [ ] 发现偏差先改 OpenSpec，再改代码。
- [ ] 每完成一个任务更新 `tasks.md`。
- [ ] DB、权限、日志、外部接口变化已补充。
- [ ] 涉及接口、权限、输入、密钥、敏感数据时，已补充安全设计。
- [ ] 涉及 MySQL 写入时，已补充 SQL、影响行数、确认方式和回滚方案。
- [ ] 涉及 `CREATE` / `ALTER` 时，已补充完整 DDL、锁表风险、回滚 SQL 和确认记录。
- [ ] 发现影响面超出预期、核心事务变更、数据迁移、停机窗口或安全漏洞时，已暂停并升级。
- [ ] 每个实现任务都能对应 PRD / OpenSpec。
- [ ] 实现后已复查 CodeGraph impact，或说明无需复查原因。
- [ ] 行为变更已规划单测/集成测试/手动验证。
- [ ] 进入实现前已完成分段审核风险等级判断。

## 完成后

- [ ] 验收标准逐条验证。
- [ ] 单测/集成测试通过。
- [ ] 只能手动验证的场景已记录环境、步骤、预期、实际和证据。
- [ ] 涉及 MySQL 时，已跑 DB Guard 检查。
- [ ] 进入 PR 前已在 `PR评审清单.md` 填写分段审核基础判断；B 类、A 类或大重构已按条件补扩展字段。
- [ ] 已执行 OpenSpec verify。
- [ ] 合并后执行 OpenSpec archive。
- [ ] 已记录实际耗时、Review 轮次、测试发现问题、回退原因、CodeGraph 定位准确性。
