# 重构检查清单

## 是否为大重构

- [ ] 跨 3 个以上模块。
- [ ] 影响 Controller / Service / Mapper 多层。
- [ ] 改核心领域模型、核心表、核心接口。
- [ ] 需要迁移大量调用方。
- [ ] 需要删除旧架构或替换框架。
- [ ] 预计超过 2 天。
- [ ] 无法一个 PR 安全合并。

任一命中，按大重构流程处理。

## Discovery

- [ ] 已用 CodeGraph 查入口。
- [ ] 已查 callers/callees。
- [ ] 已查 impact。
- [ ] 已列影响接口。
- [ ] 已列影响 Service/Mapper。
- [ ] 已列影响 DB。
- [ ] 已列权限/安全点。
- [ ] 已列测试缺口。

## RFC

- [ ] 已写 `docs/refactor/<重构名>/重构RFC.md`。
- [ ] 已说明背景和目标。
- [ ] 已说明不做什么。
- [ ] 已说明当前架构和目标架构。
- [ ] 已说明兼容策略。
- [ ] 已说明数据迁移策略。
- [ ] 已说明测试基线。
- [ ] 已说明回滚方案。

## Phase

- [ ] 已拆成多个 OpenSpec changes。
- [ ] 每个 phase 可独立合并。
- [ ] 每个 phase 可独立回滚。
- [ ] 每个 phase 有明确验收标准。
- [ ] 开发可并行的 phase 已标明依赖关系。
- [ ] 合并顺序已定义，必须串行合并。
- [ ] phase-n 的测试基线是 phase-(n-1) 合并后的代码。
- [ ] 每个 phase 的回滚方案已在测试环境验证，或说明无法验证原因。
- [ ] Feature flag 可快速切换，若适用。

## PR

- [ ] PR 只覆盖一个 phase 或 phase 内一个子任务。
- [ ] PR 链接 Asana Epic。
- [ ] PR 链接 RFC。
- [ ] PR 链接 OpenSpec change-id。
- [ ] PR 前跑 CodeGraph impact。
- [ ] PR 前跑测试策略。
- [ ] PR 前跑 Quality Gate。
- [ ] PR 描述包含回滚验证结果。
